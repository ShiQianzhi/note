# 自然语言处理

## 什么是自然语言处理？

**自然语言处理（Natural Language Processing，NLP）** 是人工智能的一个分支，旨在让计算机理解、分析、生成和翻译人类语言。

### NLP的目标

让计算机能够：
- **理解**：读懂文本的含义
- **生成**：写出通顺的文本
- **翻译**：在不同语言之间转换
- **对话**：与人进行自然对话

### 应用场景

- 聊天机器人（如ChatGPT、Siri）
- 机器翻译（如Google翻译、DeepL）
- 文本分类（如情感分析、垃圾邮件识别）
- 信息抽取（如命名实体识别、关系抽取）
- 文本摘要（如自动生成新闻摘要）

## 文本预处理

### 为什么需要预处理？

原始文本数据通常存在噪声，需要进行预处理才能被模型使用。

### 预处理步骤

#### 1. 分词（Tokenization）

将文本分割成单词、字符或子词（Subword）。

**英文分词**：
```
输入："I love natural language processing"
输出：["I", "love", "natural", "language", "processing"]
```

**中文分词**：
```
输入："我爱自然语言处理"
输出：["我", "爱", "自然语言", "处理"]
```

#### 2. 清洗（Cleaning）

- 去除标点符号
- 去除数字
- 去除特殊字符
- 转换为小写

#### 3. 停用词去除（Stopword Removal）

停用词是指频率很高但信息量很少的词：
- 英文：the, a, is, are, of, in...
- 中文：的, 是, 在, 有, 和...

#### 4. 词干提取（Stemming）

将单词还原为词干：
```
running → run
running → run
```

#### 5. 词性标注（Part-of-Speech Tagging）

为每个词标注词性：
```
I/PRP love/VBP natural/JJ language/NN processing/NN
```

## 词向量

### 为什么需要词向量？

计算机无法直接处理文本，需要将文本转换为数值向量。词向量就是将单词映射到低维实数空间的技术。

### 独热编码（One-Hot Encoding）

最简单的词向量表示方法：

```
词汇表：["猫", "狗", "鱼"]
"猫" → [1, 0, 0]
"狗" → [0, 1, 0]
"鱼" → [0, 0, 1]
```

**缺点**：
- 维度爆炸（词汇表大小即为维度）
- 无法表达语义关系（"猫"和"狗"都是[1,0,0]和[0,1,0]，没有相似性）

### 分布式表示（Distributed Representation）

将单词表示为低维稠密向量，相似的单词在向量空间中距离较近：

```
"猫" → [0.1, 0.5, -0.3, 0.7, ...]
"狗" → [0.2, 0.4, -0.2, 0.6, ...]
"鱼" → [-0.1, 0.1, 0.8, -0.2, ...]
```

### Word2Vec

Word2Vec是Google在2013年提出的词向量学习算法，有两种训练方式：

#### 1. CBOW（Continuous Bag of Words）

根据上下文预测中心词：
```
输入：["我", "爱", "语言", "处理"]
输出："自然"
```

#### 2. Skip-gram

根据中心词预测上下文：
```
输入："自然"
输出：["我", "爱", "语言", "处理"]
```

### GloVe（Global Vectors for Word Representation）

GloVe是Stanford在2014年提出的词向量算法，结合了全局统计信息。

### 词向量的特性

训练好的词向量具有以下特性：

1. **语义相似性**：相似的词距离近
   ```
   cosine("猫", "狗") ≈ 0.8
   cosine("猫", "鱼") ≈ 0.3
   ```

2. **类比推理**：可以进行数学运算
   ```
   "国王" - "男人" + "女人" ≈ "女王"
   "北京" - "中国" + "日本" ≈ "东京"
   ```

## 循环神经网络与序列建模

### 为什么需要RNN？

文本是序列数据，前后单词之间存在依赖关系，RNN能够捕捉这种依赖。

### RNN结构

```
输入序列：x₁, x₂, ..., xₙ
隐藏状态：hₜ = σ(Wₓh xₜ + Wₕh hₜ₋₁ + bₕ)
输出：yₜ = σ(Wₕy hₜ + bᵧ)
```

### 双向RNN（Bidirectional RNN）

同时考虑正向和反向的上下文信息：

```
hₜ→ = σ(Wₓh→ xₜ + Wₕh→ hₜ₋₁→ + bₕ→)
hₜ← = σ(Wₓh← xₜ + Wₕh← hₜ₊₁← + bₕ←)
hₜ = [hₜ→, hₜ←]
```

## Transformer架构

### Transformer的诞生

2017年，Google发表论文《Attention Is All You Need》，提出了Transformer架构，彻底改变了NLP领域。

### 核心思想

Transformer完全基于注意力机制（Attention Mechanism），摒弃了RNN的递归结构。

### 注意力机制

#### 什么是注意力？

注意力机制让模型能够关注输入的重要部分，类似于我们阅读时会重点关注关键信息。

#### 缩放点积注意力（Scaled Dot-Product Attention）

```
Attention(Q, K, V) = softmax(QKᵀ / √dₖ)V
```

其中：
- **Q（Query）**：查询向量，当前位置的表示
- **K（Key）**：键向量，所有位置的表示
- **V（Value）**：值向量，所有位置的实际内容
- **dₖ**：键向量的维度，用于缩放

**直观理解**：
1. 计算每个位置与当前位置的相关性（Q·K）
2. 通过softmax转换为权重
3. 根据权重对V进行加权求和

#### 多头注意力（Multi-Head Attention）

将Q、K、V投影到多个子空间，并行计算多次注意力：

```
headᵢ = Attention(QWᵠᵢ, KWₖᵢ, VWᵥᵢ)
MultiHead = concat(head₁, head₂, ..., headₕ)Wᵒ
```

**优点**：模型可以同时关注不同位置和不同类型的信息。

### Transformer结构

#### 编码器（Encoder）

编码器由N个相同的层组成，每层包含：
1. **多头自注意力机制**（Multi-Head Self-Attention）
2. **前馈神经网络**（Feed-Forward Network）
3. **残差连接**（Residual Connection）和**层归一化**（Layer Normalization）

#### 解码器（Decoder）

解码器由N个相同的层组成，每层包含：
1. **掩码多头自注意力机制**（Masked Multi-Head Self-Attention）
2. **多头注意力机制**（对编码器输出）
3. **前馈神经网络**
4. **残差连接**和**层归一化**

#### 输入嵌入和位置编码

1. **词嵌入**：将单词转换为向量
2. **位置编码**：Transformer没有递归结构，需要显式添加位置信息

```
PE(pos, 2i) = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))
```

### Transformer的优势

1. **并行计算**：不像RNN需要顺序计算，可以并行处理所有位置
2. **长距离依赖**：注意力机制可以直接捕捉任意位置之间的关系
3. **灵活建模**：多头注意力可以学习不同类型的关系

## 预训练语言模型

### 什么是预训练语言模型？

预训练语言模型是在大规模文本数据上预先训练好的模型，可以在各种NLP任务上进行微调（Fine-tuning）。

### 里程碑模型

#### 1. ELMo（2018年）

基于双向LSTM的预训练模型，首次证明预训练在NLP任务上的有效性。

#### 2. BERT（2018年）

Google提出的基于Transformer的预训练模型：

**预训练任务**：
- **掩码语言模型（Masked Language Model，MLM）**：随机掩盖部分单词，让模型预测被掩盖的单词
- **下一句预测（Next Sentence Prediction，NSP）**：让模型判断两个句子是否连续

**特点**：
- 双向Transformer
- 在11个NLP任务上刷新了SOTA

#### 3. GPT系列（2018年至今）

OpenAI提出的生成式预训练模型：

**GPT-1**：
- 12层Transformer解码器
- 预训练任务：语言模型（预测下一个单词）

**GPT-2**：
- 15亿参数
- 零样本学习（Zero-shot Learning）

**GPT-3**：
- 1750亿参数
- 强大的少样本学习（Few-shot Learning）能力

**GPT-4**：
- 多模态能力（支持图像输入）
- 更强的推理能力

#### 4. T5（2019年）

Google提出的统一框架，将所有NLP任务转换为文本到文本（Text-to-Text）格式：

```
翻译："translate English to French: Hello" → "Bonjour"
分类："sst2 sentiment: This movie is great" → "positive"
摘要："summarize: ..." → "..."
```

### 微调（Fine-tuning）

在预训练模型基础上，使用特定任务的数据进行微调：

1. 在预训练模型顶部添加任务特定的层
2. 使用较小的学习率进行训练
3. 冻结部分预训练层（可选）

### 提示工程（Prompt Engineering）

通过设计合适的提示（Prompt），让预训练模型完成特定任务：

```
Prompt: "将以下句子翻译成中文：Hello, how are you?"
Response: "你好，你怎么样？"
```

## NLP核心任务

### 文本分类（Text Classification）

将文本分类到预定义的类别：
- 情感分析（积极/消极/中性）
- 垃圾邮件识别（垃圾/正常）
- 主题分类（体育/科技/娱乐）

### 命名实体识别（Named Entity Recognition，NER）

识别文本中的命名实体：
- 人名（PER）：张三、李四
- 地名（LOC）：北京、上海
- 机构名（ORG）：清华大学、阿里巴巴

### 关系抽取（Relation Extraction）

识别实体之间的关系：
```
"张三在阿里巴巴工作" → 张三 → 工作于 → 阿里巴巴
```

### 文本生成（Text Generation）

生成符合语境的文本：
- 对话生成（聊天机器人）
- 文章生成（新闻、故事）
- 代码生成

### 机器翻译（Machine Translation）

将文本从一种语言翻译到另一种语言：
- 英译中
- 中译英
- 多语言翻译

### 文本摘要（Text Summarization）

生成文本的摘要：
- 抽取式摘要（从原文中抽取关键句子）
- 生成式摘要（生成新的摘要文本）

### 问答系统（Question Answering）

根据上下文回答问题：
- 抽取式问答（从文本中抽取答案）
- 生成式问答（生成答案）

## 评估指标

### 文本分类指标

- **准确率（Accuracy）**
- **精确率（Precision）**
- **召回率（Recall）**
- **F1分数（F1-Score）**

详见：[机器学习基础.md](机器学习基础.md)

### 文本生成指标

#### 困惑度（Perplexity，PPL）

衡量语言模型预测样本的能力：

```
PPL = exp(-(1/N) * Σlog p(xᵢ|x₁,...,xᵢ₋₁))
```

**特点**：PPL越低，模型效果越好。

#### BLEU分数（Bilingual Evaluation Understudy）

衡量机器翻译质量：

```
BLEU = BP * exp(Σwₙ * log pₙ)
```

其中：
- **pₙ**：n-gram匹配度
- **BP**：简短惩罚（如果翻译太短会扣分）

#### ROUGE分数（Recall-Oriented Understudy for Gisting Evaluation）

衡量文本摘要质量：
- ROUGE-1：单字匹配
- ROUGE-2：双字匹配
- ROUGE-L：最长公共子序列
