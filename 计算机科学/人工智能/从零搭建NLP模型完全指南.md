# 从零搭建NLP模型完全指南

## 写在前面

这篇文章会带你从零开始，一步步搭建一个完整的 NLP（自然语言处理）模型。我们以**文本分类**（判断一句话是正面还是负面情感）为例，从最简单的词袋模型一路讲到 Transformer，每一步都讲清楚"为什么这么做"和"具体怎么做"。

**目标读者**：有一点点编程基础，对 AI 感兴趣但不知道从何下手的小白。

**你将学到**：
- 文本怎么变成数字（计算机只认识数字）
- 怎么用最简单的方法做文本分类
- 词向量到底是什么、怎么工作
- 神经网络是怎么处理文本的
- Transformer 和注意力机制到底是怎么回事
- 怎么用预训练模型快速搭建好用的 NLP 系统

## 第一章：整体思路——我们到底在做什么？

### 1.1 NLP 模型的本质

NLP 模型的本质就是一个**函数**：

```
输入：一句中文（如"这部电影真好看"）
输出：一个标签（如"正面情感"）
```

但计算机不认识中文，它只认识数字。所以我们的核心任务是：

> **把文字变成数字，然后用数学模型去拟合规律。**

### 1.2 搭建 NLP 模型的完整流程

```
第1步：明确任务 → 第2步：准备数据 → 第3步：文本预处理
     → 第4步：选择模型 → 第5步：训练模型 → 第6步：评估效果
```

我们以**情感分析**（判断一句话是正面还是负面）为例，一步步走完整条路。

## 第二章：数据准备——一切的开始

### 2.1 数据长什么样？

做文本分类，我们需要的数据集长这样：

| 文本 | 标签 |
|------|------|
| 这部电影真好看，太感动了！ | 正面 |
| 浪费时间，一点都不好看 | 负面 |
| 演员演技很棒，剧情也不错 | 正面 |
| 烂片，看了一半就走了 | 负面 |
| ... | ... |

每一行是一个样本，包含：
- **文本（Text）**：输入的句子
- **标签（Label）**：对应的分类（正面/负面）

### 2.2 数据集怎么划分？

拿到数据后，我们要把它分成三份：

| 数据集 | 占比 | 用途 |
|--------|------|------|
| 训练集（Training Set） | 70%~80% | 用来训练模型 |
| 验证集（Validation Set） | 10%~15% | 用来调参数、选模型 |
| 测试集（Test Set） | 10%~15% | 用来最终评估效果 |

**为什么要分三份？**
- 训练集是"课本"，模型从中学知识
- 验证集是"模拟考试"，用来检验学习效果、调整方法
- 测试集是"期末考试"，只考一次，用来衡量最终水平

> 📌 **重要原则**：测试集只能用一次！如果反复用测试集调参，就相当于提前知道了考试题，成绩会虚高。

## 第三章：文本预处理——把文字变成数字

### 3.1 为什么要预处理？

计算机只认识数字，不认识"猫""狗""你好"这些文字。我们需要想办法把文字转换成数字。

### 3.2 第一步：分词（Tokenization）

分词就是把一句话拆成一个个"词"。

**英文分词**比较简单，用空格分隔就行：
```
输入："I love natural language processing"
输出：["I", "love", "natural", "language", "processing"]
```

**中文分词**麻烦一点，因为中文词之间没有空格：
```
输入："我爱自然语言处理"
输出：["我", "爱", "自然语言", "处理"]
```

> 📌 **术语**：拆分后的每一个单元叫做 **Token**（标记），可以是词、字、或者子词（Subword）。

### 3.3 第二步：构建词表（Vocabulary）

把数据集中所有出现过的词收集起来，给每个词分配一个唯一的编号：

```
词表：{
    "我": 0,
    "爱": 1,
    "电影": 2,
    "好看": 3,
    "这部": 4,
    "真": 5,
    ...
}
```

词表的大小记为 **V**（Vocabulary Size），通常是几万到几十万。

### 3.4 第三步：把句子转换成编号序列

有了词表，就可以把每个词换成它的编号：

```
句子："这部 电影 真 好看"
编号：[4, 2, 5, 3]
```

现在文本变成了数字序列，但还不够——这些编号只是索引，没有语义信息。

## 第四章：入门级模型——词袋模型 + 逻辑回归

我们从最简单的方法开始：**词袋模型（Bag of Words，BOW）**。

### 4.1 词袋模型是什么？

词袋模型的思路非常简单：**不考虑词的顺序，只统计每个词出现了多少次**。

就像把一句话的词都扔进一个袋子里，搅匀了再数。

举个例子：

```
句子："这部 电影 真 好看"
词表：["我", "爱", "电影", "好看", "这部", "真", "烂", "片"]

词袋向量：[0, 0, 1, 1, 1, 1, 0, 0]
```

每个位置对应词表中的一个词，值是这个词出现的次数。

### 4.2 词袋模型的问题

```
句子A："我喜欢这部电影，不讨厌"
句子B："我讨厌这部电影，不喜欢"
```

这两句话用词袋模型表示，向量是**完全一样的**！因为词的种类和数量相同，只是顺序不同。

> 📌 **词袋模型的致命缺陷**：丢失了语序信息，而语序对语义至关重要。

但别急，虽然简单，词袋模型在一些简单任务上效果还不错，而且非常好理解，是入门的好起点。

### 4.3 升级版：TF-IDF

TF-IDF 是词袋模型的改进版，它不光统计词频，还考虑词的"重要性"。

#### 什么是 TF？

**TF（Term Frequency，词频）**：某个词在文档中出现的频率。

```
TF = 词在文档中出现的次数 / 文档总词数
```

出现次数越多，这个词越能代表这篇文档。

#### 什么是 IDF？

**IDF（Inverse Document Frequency，逆文档频率）**：衡量一个词的"稀有程度"。

```
IDF = log(总文档数 / (包含该词的文档数 + 1))
```

- 如果一个词在很多文档中都出现（比如"的""是"），说明它没什么区分度，IDF 值小
- 如果一个词只在少数文档中出现，说明它很有特点，IDF 值大

#### TF-IDF 怎么算？

```
TF-IDF = TF × IDF
```

**直观理解**：一个词在当前文档中出现得多，在其他文档中出现得少，它的 TF-IDF 值就大，说明它对这篇文档很重要。

> 📌 **为什么叫"逆"文档频率？** 因为文档频率越高，这个值越小，所以是"逆"过来的。

### 4.4 用逻辑回归做分类

有了 TF-IDF 向量，我们就可以用它来做分类了。最简单的分类器是**逻辑回归（Logistic Regression）**。

#### 逻辑回归在做什么？

逻辑回归就是给每个特征（每个词）分配一个权重，然后加权求和，最后用 sigmoid 函数映射到 0~1 之间，当作概率。

```
z = w₁x₁ + w₂x₂ + ... + wₙxₙ + b
y = σ(z) = 1 / (1 + e^(-z))
```

其中：
- `xᵢ`：第 i 个词的 TF-IDF 值
- `wᵢ`：第 i 个词的权重（模型要学习的参数）
- `b`：偏置（bias）
- `σ`：sigmoid 函数，把任何实数映射到 (0, 1) 区间

**举个例子**：
- 如果"好看"的权重 w = 2.5，说明这个词越多，越可能是正面评价
- 如果"烂片"的权重 w = -3.0，说明这个词越多，越可能是负面评价

#### sigmoid 函数长什么样？

```
      1.0 ┤        ╭──────
          │       ╱
      0.5 ┼──────╯
          │    ╱
      0.0 ┼────
          └────────────
          -3    0    3
```

- 当 z 很大时，σ(z) 接近 1（很确定是正面）
- 当 z 很小时，σ(z) 接近 0（很确定是负面）
- 当 z = 0 时，σ(z) = 0.5（不确定）

### 4.5 模型怎么"学"？——损失函数和梯度下降

#### 损失函数（Loss Function）

我们怎么知道模型学得好不好？用**损失函数**来衡量。

对于二分类任务，最常用的是**交叉熵损失（Cross-Entropy Loss）**：

```
Loss = -y * log(ŷ) - (1 - y) * log(1 - ŷ)
```

其中：
- `y`：真实标签（1=正面，0=负面）
- `ŷ`：模型预测的概率

**直观理解**：
- 如果真实是正面（y=1），预测概率 ŷ 越接近 1，损失越小
- 如果真实是负面（y=0），预测概率 ŷ 越接近 0，损失越小
- 预测错了，损失就大；预测对了，损失就小

#### 梯度下降（Gradient Descent）

有了损失函数，怎么让损失变小？用**梯度下降**。

梯度下降的思路很简单：
1. 随便给权重 w 一个初始值
2. 计算损失函数对 w 的梯度（导数）
3. 沿着梯度的反方向更新 w
4. 重复直到损失不再下降

```
w_new = w_old - α * dLoss/dw
```

其中：
- `α`：学习率（Learning Rate），控制每一步走多大
- `dLoss/dw`：损失函数对 w 的导数

> 📌 **梯度下降就像下山**：你站在山坡上，往最陡的方向走一步，再看一下方向，再走一步，直到走到谷底。

## 第五章：进阶级——词向量（Word Embedding）

词袋模型太简单了，丢失了太多信息。我们需要更好的文本表示方法，这就是**词向量**。

### 5.1 什么是词向量？

词向量就是把每个词表示成一个**低维稠密向量**（比如 100 维、300 维）。

```
"猫" → [0.12, -0.34, 0.56, ..., 0.78]  （300个小数）
"狗" → [0.15, -0.31, 0.52, ..., 0.75]  （300个小数）
```

**为什么叫"稠密"？** 因为和词袋模型的稀疏向量（大部分是0）比起来，词向量的每个维度通常都有非零值。

### 5.2 词向量好在哪里？

#### 好处一：语义相似的词，向量也相似

```
cos("猫", "狗") ≈ 0.85   （很相似）
cos("猫", "汽车") ≈ 0.1  （不相似）
```

**余弦相似度（Cosine Similarity）**：

```
cos(v₁, v₂) = (v₁ · v₂) / (||v₁|| * ||v₂||)
```

- 值为 1：方向完全相同 → 语义非常相似
- 值为 0：方向垂直 → 语义无关
- 值为 -1：方向相反 → 语义相反

#### 好处二：可以做类比推理

```
"国王" - "男人" + "女人" ≈ "女王"
```

这是因为词向量在不同维度上编码了不同的语义特征（比如性别、职业、大小等）。

### 5.3 词向量是怎么学来的？

最经典的算法是 **Word2Vec**，它有两种训练方式：CBOW 和 Skip-gram。

#### Skip-gram：用中心词预测上下文

**思路**：给定中心词，预测它周围的词出现的概率。

比如句子"我 爱 自然语言处理 很有趣"，窗口大小是2：
- 中心词是"自然语言处理"
- 上下文是"爱"和"很有趣"
- 模型要学：给定"自然语言处理"，"爱"出现的概率有多大？

**数学表达**：

```
P(context | center) = exp(v_context · v_center) / Σexp(v_word · v_center)
```

**为什么这样就能学到语义？**
- "猫"经常和"宠物""可爱""吃鱼"一起出现
- "狗"也经常和"宠物""可爱""吃肉"一起出现
- 它们的上下文很像，所以向量也会很像

#### 负采样（Negative Sampling）

上面的公式要对词表里所有词求和，词表有几万甚至几十万词，计算太慢了。

负采样的思路是：**不跟所有词比，只随机选几个"坏例子"比**。

损失函数变成：

```
Loss = -log σ(v_c · v_w) - Σlog σ(-v_n · v_w)
```

其中：
- `v_c`：真实上下文词的向量（正样本）
- `v_n`：随机选的词的向量（负样本）
- `σ`：sigmoid 函数

**直观理解**：
- 第一项：让中心词和真实上下文词的向量点积变大（拉近）
- 第二项：让中心词和随机词的向量点积变小（推远）

### 5.4 怎么用词向量表示句子？

有了词向量，怎么表示一整句话呢？

#### 方法一：平均池化（Average Pooling）

最简单的方法：把所有词向量加起来取平均。

```
句子向量 = (v_我 + v_爱 + v_电影 + v_好看) / 4
```

**优点**：简单
**缺点**：丢失了语序信息（和词袋模型一样的毛病）

#### 方法二：深度学习模型

用 RNN、CNN、Transformer 等更复杂的模型来处理序列，保留语序信息。

我们下一章节详细讲。

## 第六章：神经网络篇——用深度学习处理文本

### 6.1 从逻辑回归到全连接神经网络

逻辑回归只有一层，全连接神经网络（FNN）有很多层：

```
输入层 → 隐藏层1 → 隐藏层2 → ... → 输出层
```

每一层的计算方式是：

```
z = Wx + b
a = σ(z)
```

和逻辑回归一样，只是多了几层。

**为什么多层有用？**
- 第一层学习简单的模式
- 第二层组合简单模式，学习复杂一点的模式
- 层数越多，能学习的模式越复杂

> 📌 **万能近似定理**：一个足够宽的单隐藏层神经网络，理论上可以拟合任何连续函数。但"足够宽"可能宽到不现实，所以实际上我们用"深"的网络。

### 6.2 前向传播（Forward Propagation）

数据从输入层流向输出层的过程叫前向传播。

以两层网络为例：

```
z⁽¹⁾ = W⁽¹⁾x + b⁽¹⁾      （第一层加权求和）
a⁽¹⁾ = ReLU(z⁽¹⁾)         （第一层激活）
z⁽²⁾ = W⁽²⁾a⁽¹⁾ + b⁽²⁾    （第二层加权求和）
ŷ = sigmoid(z⁽²⁾)          （输出层）
```

#### ReLU 是什么？

ReLU（Rectified Linear Unit）是最常用的激活函数：

```
ReLU(x) = max(0, x)
```

- x > 0 时，输出 x
- x ≤ 0 时，输出 0

> 📌 **为什么需要激活函数？** 如果没有激活函数，多层网络就退化成单层了（线性变换的组合还是线性变换）。激活函数引入了非线性，让网络能学习复杂的模式。

### 6.3 反向传播（Backpropagation）

模型怎么学习？和逻辑回归一样，用梯度下降。但多层网络怎么算梯度？用**反向传播**。

#### 链式法则

反向传播的数学基础是**链式法则**：

```
dLoss/dW⁽¹⁾ = dLoss/da⁽¹⁾ * da⁽¹⁾/dz⁽¹⁾ * dz⁽¹⁾/dW⁽¹⁾
```

从后往前一层一层算：
1. 先算输出层的误差
2. 再算倒数第二层的误差
3. 一直往前算到第一层
4. 每一层都更新参数

> 📌 **反向传播就像走迷宫**：前向传播是从起点走到终点，看看有多远（损失是多少）；反向传播是从终点往回走，标记哪条路贡献大、哪条路贡献小，然后调整。

### 6.4 循环神经网络（RNN）——处理序列数据

全连接网络有个问题：它把文本当成了一个无序的向量包，丢失了语序。

**循环神经网络（Recurrent Neural Network，RNN）** 专门处理序列数据。

#### RNN 怎么工作？

RNN 的核心是**隐藏状态（Hidden State）**，它就像网络的"记忆"。

```
h₀ → h₁ → h₂ → h₃ → ... → hₙ
      ↑    ↑    ↑          ↑
     x₁   x₂   x₃         xₙ
```

每个时刻的计算：

```
hₜ = tanh(Wₕₕ · hₜ₋₁ + Wₓₕ · xₜ + bₕ)
yₜ = Wₕᵧ · hₜ + bᵧ
```

其中：
- `hₜ`：第 t 个时刻的隐藏状态（记忆）
- `xₜ`：第 t 个时刻的输入（词向量）
- `hₜ₋₁`：上一个时刻的隐藏状态（之前的记忆）

**直观理解**：RNN 读一个词，更新一下记忆；再读一个词，再更新一下记忆。读到最后，记忆里就包含了整句话的信息。

#### RNN 的问题：梯度消失

RNN 有个大毛病：**记不住很久以前的信息**。

比如这句话：
```
我出生在法国，...（中间有100个词）...，我能流利地说____。
```

要填"法语"，需要记得开头的"法国"。但 RNN 处理到第100个词时，开头的信息已经"稀释"得差不多了。

**为什么会这样？** 因为反向传播时，梯度每经过一层就会乘以一个小于1的数，乘几十上百次后，梯度就变得几乎为0了——这叫**梯度消失（Vanishing Gradient）**。

### 6.5 LSTM——给 RNN 装个"长期记忆"

**LSTM（Long Short-Term Memory，长短期记忆网络）** 是 RNN 的改进版，专门解决梯度消失问题。

#### LSTM 的核心：记忆细胞（Cell State）

LSTM 多了一条"传送带"——记忆细胞 cₜ：

```
c₀ → c₁ → c₂ → c₃ → ... → cₙ
```

信息可以直接在传送带上流动，不容易丢失。

#### 三个门（Gate）

LSTM 用三个门来控制记忆细胞：

1. **遗忘门（Forget Gate）**：决定从记忆中丢弃什么
   ```
   fₜ = σ(Wf · [hₜ₋₁, xₜ] + bf)
   ```

2. **输入门（Input Gate）**：决定往记忆中存入什么
   ```
   iₜ = σ(Wi · [hₜ₋₁, xₜ] + bi)
   c̃ₜ = tanh(Wc · [hₜ₋₁, xₜ] + bc)
   cₜ = fₜ * cₜ₋₁ + iₜ * c̃ₜ
   ```

3. **输出门（Output Gate）**：决定输出什么
   ```
   oₜ = σ(Wo · [hₜ₋₁, xₜ] + bo)
   hₜ = oₜ * tanh(cₜ)
   ```

**直观理解**：
- 遗忘门："之前的记忆里，哪些不重要了，可以忘掉？"
- 输入门："新来了一些信息，哪些值得记下来？"
- 输出门："现在该输出什么？"

> 📌 **门为什么用 sigmoid？** 因为 sigmoid 输出在 0~1 之间，0 表示"完全不放行"，1 表示"完全放行"，正好适合当"门"。

### 6.6 双向 LSTM（BiLSTM）

单向 RNN 只能"看到"前面的词，看不到后面的词。但有时候要理解一个词，需要同时看上下文。

**双向 LSTM** 就是跑两个 LSTM：
- 一个从左往右读（正向）
- 一个从右往左读（反向）
- 两个方向的隐藏状态拼起来用

```
正向：h₀→ → h₁→ → h₂→ → ... → hₙ→
反向：h₀← ← h₁← ← h₂← ← ... ← hₙ←

最终：hₜ = [hₜ→, hₜ←]
```

### 6.7 TextCNN——用 CNN 做文本分类

你可能以为 CNN 只能处理图像，但其实 CNN 也能处理文本！

**TextCNN** 的思路很简单：把一维的文本序列当作一维的"图像"，用一维卷积来提取特征。

#### 卷积怎么作用在文本上？

假设词向量维度是 d，卷积核大小是 k × d（覆盖 k 个词，每个词的 d 维全看）：

```
卷积核在句子上滑动，每次覆盖 k 个词，
做一次加权求和，得到一个值。
滑完一整句，得到一个特征向量。
```

用多个不同大小的卷积核，可以提取不同粒度的特征：
- 2-gram 卷积核：提取两个词的组合特征（如"不好看"）
- 3-gram 卷积核：提取三个词的组合特征
- ...

**为什么 TextCNN 有效？** 因为很多时候，关键词和短语（n-gram）就足以判断情感了，不需要完整的语序。

## 第七章：Transformer 篇——注意力即一切

2017 年 Google 发表论文《Attention Is All You Need》，提出了 **Transformer** 架构，彻底改变了 NLP 领域。

### 7.1 RNN 的问题

RNN（包括 LSTM）有两个大问题：

1. **没法并行计算**：必须先算 h₁，再算 h₂，再算 h₃... 是串行的
2. **长距离依赖还是弱**：虽然 LSTM 比 RNN 好，但距离太远还是容易忘

Transformer 用**注意力机制（Attention Mechanism）** 解决了这两个问题。

### 7.2 注意力机制到底是什么？

注意力机制的灵感来自人类的视觉：我们看东西时，会把注意力集中在重要的地方，忽略不重要的地方。

在 NLP 中，注意力就是：**处理一个词的时候，动态地决定该关注句子中的哪些词。**

#### 一个直观的例子

句子："自然语言处理是人工智能的一个分支，它发展很快。"

要理解"它"指的是什么，我们需要把注意力集中在"自然语言处理"上。

### 7.3 缩放点积注意力（Scaled Dot-Product Attention）

Transformer 中最基础的注意力机制叫"缩放点积注意力"。

首先，每个词有三种向量：
- **Q（Query，查询）**：我想找什么
- **K（Key，键）**：我有什么
- **V（Value，值）**：我的实际内容

这三个向量都是从词向量通过不同的线性变换得到的：

```
Q = X · Wq
K = X · Wk
V = X · Wv
```

然后，注意力计算分三步：

**第一步：计算注意力得分**

每个查询 Q 跟所有的键 K 做点积，得到注意力分数：

```
S = Q · Kᵀ
```

分数越大，表示两个词越相关。

**第二步：缩放 + Softmax**

先缩放（除以 √dₖ，防止点积太大导致 softmax 饱和），再用 softmax 转换成概率：

```
A = softmax(S / √dₖ)
```

A 是一个矩阵，A[i][j] 表示第 i 个词对第 j 个词的注意力权重。

**第三步：加权求和**

用注意力权重对 V 加权求和，得到最终的表示：

```
O = A · V
```

**最终公式**：

```
Attention(Q, K, V) = softmax(QKᵀ / √dₖ)V
```

> 📌 **为什么要除以 √dₖ？** 因为向量维度 dₖ 很大时，点积的方差会很大，导致 softmax 输出接近 0 或 1，梯度会很小。除以 √dₖ 可以把方差拉回到 1，让训练更稳定。

### 7.4 多头注意力（Multi-Head Attention）

多头注意力就是把 Q、K、V 投影到多个子空间，分别计算注意力，再拼起来。

```
把 Q、K、V 分成 h 份（h 个头）
每个头各自计算 Attention
把结果拼起来，再做一次线性变换
```

**为什么多头有用？** 因为不同的头可以关注不同类型的关系：
- 有的头关注语法关系（主谓宾）
- 有的头关注指代关系（它指什么）
- 有的头关注局部邻近的词
- ...

### 7.5 Transformer 的完整结构

Transformer 是**编码器-解码器（Encoder-Decoder）** 架构。

#### 编码器（Encoder）

编码器由 N 层相同的层堆叠而成，每层包含：

1. **多头自注意力（Multi-Head Self-Attention）**
   - 自己跟自己做注意力，捕捉句子内部词与词的关系
2. **前馈神经网络（Feed-Forward Network，FFN）**
   - 每个位置独立通过一个两层的全连接网络
3. **残差连接（Residual Connection）+ 层归一化（Layer Norm）**
   - 残差连接：x + Sublayer(x)，防止梯度消失
   - 层归一化：归一化特征分布，加速收敛

> 📌 **自注意力（Self-Attention）**：Q、K、V 都来自同一个序列，就是自注意力。自己跟自己做注意力。

#### 解码器（Decoder）

解码器也有 N 层，每层比编码器多一个注意力层：

1. **掩码多头自注意力（Masked Multi-Head Self-Attention）**
   - 加掩码，防止"看到未来的词"（生成时只能看前面的词）
2. **编码器-解码器注意力（Encoder-Decoder Attention）**
   - Q 来自解码器，K 和 V 来自编码器输出
   - 让解码器关注输入序列的相关部分
3. **前馈神经网络**
4. **残差连接 + 层归一化**

#### 位置编码（Positional Encoding）

Transformer 没有循环结构，不知道词的顺序。所以需要显式地把位置信息加进去。

Transformer 用三角函数做位置编码：

```
PE(pos, 2i) = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))
```

其中：
- `pos`：位置（第几个词）
- `i`：维度索引
- `d_model`：模型维度

**为什么用三角函数？** 因为三角函数有个性质：对于固定的偏移量 k，PE(pos+k) 可以表示成 PE(pos) 的线性函数，这让模型能方便地学习相对位置关系。

### 7.6 为什么 Transformer 这么强？

1. **可以并行计算**：所有位置同时算，不像 RNN 要串行
2. **直接建模长距离依赖**：注意力机制让任意两个词可以直接"对话"，不管离多远
3. **灵活**：多头注意力可以同时学习多种类型的关系

## 第八章：预训练模型时代——站在巨人的肩膀上

### 8.1 什么是预训练？

**预训练（Pre-training）**：先在大量通用文本上训练一个大模型，让它学到通用的语言知识；然后在具体任务上**微调（Fine-tuning）**，用少量数据就能取得很好的效果。

就像一个人先读了很多书（预训练），掌握了语言和常识，然后学某个专业（微调）就很快。

### 8.2 BERT：双向预训练的里程碑

**BERT（Bidirectional Encoder Representations from Transformers）** 是 Google 在 2018 年提出的预训练模型。

#### BERT 的预训练任务

BERT 用两个任务做预训练：

**任务一：掩码语言模型（Masked Language Model，MLM）**

随机掩盖句子中的一些词，让模型预测被掩盖的词：

```
输入：我 [MASK] 自然语言处理
目标：预测 [MASK] = "爱"
```

这就是为什么 BERT 是"双向"的——预测一个词时，可以同时看左边和右边的上下文。

**任务二：下一句预测（Next Sentence Prediction，NSP）**

给两句话，让模型判断第二句话是不是第一句话的下一句：

```
句子A：今天天气真好。
句子B：我们去公园玩吧。
标签：是下一句

句子A：今天天气真好。
句子B：猫很可爱。
标签：不是下一句
```

这个任务让模型学习句子之间的关系。

#### 怎么用 BERT 做文本分类？

1. 取 <[BOS_never_used_51bce0c785ca2f68081bfa7d91973934]> 标记的最终隐藏状态（代表整个句子的向量）
2. 在上面加一个全连接层，输出分类概率
3. 用你的数据微调（Fine-tuning）整个模型

### 8.3 GPT：生成式预训练

**GPT（Generative Pre-trained Transformer）** 是 OpenAI 提出的生成式预训练模型。

和 BERT 的区别：
- BERT 用编码器，双向，擅长理解任务
- GPT 用解码器，单向，擅长生成任务

GPT 的预训练任务很简单：**给定前面的词，预测下一个词**。

```
输入：我 爱 自然 语言
目标：预测下一个词 = "处理"
```

这叫**自回归语言模型（Autoregressive Language Model）**。

### 8.4 微调（Fine-tuning）怎么做？

预训练模型已经学到了通用语言知识，在具体任务上只需要用较小的学习率、较少的数据训练一下就行。

**为什么叫"微"调？** 因为只需要做"小调整"，不是从零开始学。

微调时的注意事项：
- 学习率要小（比如 2e-5），不然会"忘掉"预训练学到的知识
- 训练轮次不用太多（比如 3~5 轮）
- 可以冻结底层，只训顶层（可选）

## 第九章：模型评估——怎么知道模型好不好？

### 9.1 分类任务的评估指标

#### 准确率（Accuracy）

```
准确率 = 预测正确的样本数 / 总样本数
```

最简单直观，但在类别不平衡时会骗人。

#### 混淆矩阵（Confusion Matrix）

```
              预测值
            正面    负面
真实值 正面  TP      FN
      负面  FP      TN
```

- **TP（True Positive）**：实际正面，预测正面（预测对了）
- **FN（False Negative）**：实际正面，预测负面（漏了）
- **FP（False Positive）**：实际负面，预测正面（误报）
- **TN（True Negative）**：实际负面，预测负面（预测对了）

#### 精确率（Precision）

```
精确率 = TP / (TP + FP)
```

预测为正面的里面，有多少是真的正面。

**什么时候看重精确率？** 比如垃圾邮件过滤——宁可放过垃圾邮件，也别把正常邮件当垃圾邮件删了。

#### 召回率（Recall）

```
召回率 = TP / (TP + FN)
```

真正的正面中，被找出来的有多少。

**什么时候看重召回率？** 比如癌症筛查——宁可误判健康人，也别漏掉癌症患者。

#### F1 分数

```
F1 = 2 × (精确率 × 召回率) / (精确率 + 召回率)
```

精确率和召回率的调和平均，综合衡量模型。

### 9.2 怎么判断模型是好是坏？

- **训练集准确率高，测试集也高**：好，模型泛化能力强
- **训练集准确率高，测试集低**：过拟合（Overfitting），模型记牢了训练数据但不会举一反三
- **训练集准确率低，测试集也低**：欠拟合（Underfitting），模型太简单，学不会

**怎么解决过拟合？**
- 增加训练数据
- 减少模型复杂度
- 加正则化（Dropout、L2正则）
- 数据增强

**怎么解决欠拟合？**
- 增加模型复杂度（更深更大的网络）
- 增加特征
- 减少正则化

## 第十章：完整流程总结

让我们用一个完整的情感分析例子，串起所有步骤：

### 第一步：明确任务

- 任务：中文情感二分类
- 输入：一句话
- 输出：正面 / 负面

### 第二步：准备数据

- 收集 10000 条标注好的评论数据
- 按 8:1:1 划分训练集、验证集、测试集

### 第三步：文本预处理

1. 分词（用 jieba 或 BERT 自带的 tokenizer）
2. 构建词表（或用预训练模型的词表）
3. 转成 token id 序列
4. 统一长度（太短补 0，太长截断）

### 第四步：选择模型

- 简单快速：TF-IDF + 逻辑回归
- 效果好：BERT 微调

### 第五步：训练模型

1. 初始化模型参数
2. 前向传播，计算预测
3. 计算损失
4. 反向传播，计算梯度
5. 用优化器（Adam）更新参数
6. 重复，直到验证集效果不再提升

### 第六步：评估效果

- 在测试集上算准确率、精确率、召回率、F1
- 看看哪些样本预测错了，分析原因

### 第七步：上线部署

- 保存训练好的模型
- 写一个 API，接收文本，返回预测结果
- 监控线上效果，持续迭代
