# 强化学习

## 什么是强化学习？

**强化学习（Reinforcement Learning，RL）** 是机器学习的一个分支，它让智能体（Agent）通过与环境的交互来学习最优策略。

### RL的核心思想

强化学习的核心思想是**试错学习**：
1. 智能体执行一个动作
2. 环境给出反馈（奖励或惩罚）
3. 智能体根据反馈调整策略
4. 重复以上过程，直到学到最优策略

### 类比理解

想象训练一只小狗：
- 小狗执行一个动作（如坐下）
- 如果做得对，给它一块肉（奖励）
- 如果做得不对，不理它（惩罚）
- 小狗学会"坐下"可以得到奖励，以后就会更倾向于坐下

这就是强化学习的基本过程。

### 应用场景

- 游戏AI（AlphaGo、DOTA2）
- 机器人控制（自动驾驶、机械臂）
- 资源管理（电力调度、库存控制）
- 金融交易（量化交易策略）

## 强化学习基本概念

### 智能体（Agent）

智能体是学习和决策的主体：
- 接收环境状态
- 执行动作
- 根据奖励学习

### 环境（Environment）

环境是智能体交互的外部世界：
- 接收智能体的动作
- 返回新状态和奖励

### 状态（State）

状态是环境的描述：
- 完全可观察状态（Full Observable）：智能体知道环境的全部信息
- 部分可观察状态（Partially Observable）：智能体只知道环境的部分信息

### 动作（Action）

动作是智能体可以执行的操作：
- 离散动作（Discrete）：有限个选择（如上下左右）
- 连续动作（Continuous）：无限个选择（如速度、角度）

### 奖励（Reward）

奖励是环境对智能体动作的反馈：
- 正奖励：鼓励该动作
- 负奖励：惩罚该动作

### 策略（Policy）

策略是智能体根据状态选择动作的规则：
- **确定性策略**：π(a|s) = 1（确定选择某个动作）
- **随机性策略**：π(a|s) = p（以概率p选择动作a）

### 价值函数（Value Function）

价值函数衡量状态或动作的长期价值：

#### 状态价值函数（State Value Function）

从状态s出发，遵循策略π的期望回报：
```
Vπ(s) = E[Rₜ + γRₜ₊₁ + γ²Rₜ₊₂ + ... | Sₜ = s]
```

#### 动作价值函数（Action Value Function）

从状态s出发，执行动作a后遵循策略π的期望回报：
```
Qπ(s, a) = E[Rₜ + γVπ(Sₜ₊₁) | Sₜ = s, Aₜ = a]
```

其中，`γ`是折扣因子（Discount Factor），控制未来奖励的重要程度。

### 回报（Return）

回报是未来奖励的总和：
```
Gₜ = Rₜ₊₁ + γRₜ₊₂ + γ²Rₜ₊₃ + ...
```

### 探索与利用（Exploration vs Exploitation）

- **探索**：尝试新动作，可能发现更好的策略
- **利用**：选择已知的最优动作，获得最大奖励

**平衡**：需要在探索和利用之间找到平衡。

## 马尔可夫决策过程

### 什么是MDP？

**马尔可夫决策过程（Markov Decision Process，MDP）** 是强化学习的数学框架。

### MDP组成

一个MDP由以下5个元素组成：

1. **状态集合（State Space）**：S
2. **动作集合（Action Space）**：A
3. **状态转移概率（Transition Probability）**：P(s'|s, a)
4. **奖励函数（Reward Function）**：R(s, a, s')
5. **折扣因子（Discount Factor）**：γ

### 马尔可夫性质

当前状态包含了所有历史信息：
```
P(Sₜ₊₁ | Sₜ, Aₜ) = P(Sₜ₊₁ | S₀, A₀, ..., Sₜ, Aₜ)
```

## 动态规划

### 策略评估（Policy Evaluation）

给定策略π，计算价值函数Vπ(s)：
```
Vπ(s) = Σₐ π(a|s) * Σₛ' P(s'|s, a) * [R(s, a, s') + γVπ(s')]
```

### 策略改进（Policy Improvement）

根据当前价值函数改进策略：
```
π'(s) = argmaxₐ Σₛ' P(s'|s, a) * [R(s, a, s') + γVπ(s')]
```

### 策略迭代（Policy Iteration）

交替执行策略评估和策略改进，直到策略收敛。

### 值迭代（Value Iteration）

直接优化价值函数，不需要显式策略：
```
V(s) = maxₐ Σₛ' P(s'|s, a) * [R(s, a, s') + γV(s')]
```

## Monte Carlo方法

### 什么是Monte Carlo方法？

Monte Carlo方法通过采样完整的轨迹来估计价值函数。

### 首次访问MC

只使用每个状态的首次访问来估计价值：
```
V(s) = (G₁ + G₂ + ... + Gₖ) / k
```

其中，Gᵢ是第i次访问状态s后的回报。

### 每次访问MC

使用每个状态的所有访问来估计价值。

## 时序差分学习

### 什么是时序差分学习？

时序差分学习结合了动态规划和Monte Carlo的优点：
- 像动态规划一样使用自举（Bootstrapping）
- 像Monte Carlo一样使用采样

### TD(0)

使用一步预测来更新价值函数：
```
V(Sₜ) = V(Sₜ) + α[Rₜ₊₁ + γV(Sₜ₊₁) - V(Sₜ)]
```

其中，`α`是学习率，`Rₜ₊₁ + γV(Sₜ₊₁) - V(Sₜ)`是TD误差。

### n步TD

使用n步预测来更新价值函数：
```
V(Sₜ) = V(Sₜ) + α[Gₜ⁽ⁿ⁾ - V(Sₜ)]
```

其中，`Gₜ⁽ⁿ⁾ = Rₜ₊₁ + γRₜ₊₂ + ... + γⁿ⁻¹Rₜ₊ₙ + γⁿV(Sₜ₊ₙ)`。

## Q-learning

### 什么是Q-learning？

Q-learning是一种基于动作价值函数的强化学习算法。

### Q-learning更新规则

```
Q(Sₜ, Aₜ) = Q(Sₜ, Aₜ) + α[Rₜ₊₁ + γmaxₐ' Q(Sₜ₊₁, Aₜ₊₁) - Q(Sₜ, Aₜ)]
```

### 特点

- **离线策略（Off-policy）**：学习最优策略，而不依赖于当前策略
- **收敛性**：保证收敛到最优动作价值函数

### ε-贪心策略

为了平衡探索和利用，使用ε-贪心策略：
- 以概率ε随机选择动作（探索）
- 以概率1-ε选择Q值最大的动作（利用）

### Q-learning算法流程

1. 初始化Q表，Q(s, a) = 0
2. 对于每个episode：
   a. 初始化状态s
   b. 当s不是终止状态时：
      i. 使用ε-贪心策略选择动作a
      ii. 执行动作a，得到奖励r和新状态s'
      iii. 更新Q值：Q(s, a) += α[r + γmaxₐ' Q(s', a') - Q(s, a)]
      iv. 设置s = s'

## 深度Q网络（DQN）

### 为什么需要DQN？

传统Q-learning使用Q表存储值，当状态空间很大时（如游戏画面），Q表无法存储所有状态。

### DQN结构

DQN使用神经网络代替Q表：
- 输入：状态（如游戏画面）
- 输出：每个动作的Q值

### DQN创新点

#### 1. 经验回放（Experience Replay）

将经验（s, a, r, s'）存储在回放缓冲区中，随机采样训练：
- 打破样本之间的相关性
- 提高数据利用率

#### 2. 目标网络（Target Network）

使用两个网络：
- **当前网络**：用于选择动作和计算Q值
- **目标网络**：用于计算目标Q值，定期更新

**好处**：稳定训练过程

### DQN算法流程

1. 初始化当前网络Q和目标网络Q'
2. 初始化经验回放缓冲区D
3. 对于每个episode：
   a. 初始化状态s
   b. 当s不是终止状态时：
      i. 使用ε-贪心策略选择动作a
      ii. 执行动作a，得到奖励r和新状态s'
      iii. 将经验（s, a, r, s', done）存入D
      iv. 从D中随机采样一批经验
      v. 计算目标Q值：yᵢ = rᵢ + γmaxₐ' Q'(s'ᵢ, a')
      vi. 更新当前网络：最小化(yᵢ - Q(sᵢ, aᵢ))²
      vii. 定期更新目标网络：Q' = Q

### DQN变体

- **Double DQN**：解决过估计问题
- **Dueling DQN**：分离状态价值和动作优势
- **Prioritized Experience Replay**：优先采样重要经验

## 策略梯度

### 什么是策略梯度？

策略梯度直接优化策略函数，而不是价值函数。

### 策略梯度定理

```
∇J(θ) = E[∇logπθ(a|s) * G]
```

其中，`θ`是策略网络的参数，`G`是回报。

### REINFORCE算法

REINFORCE是最简单的策略梯度算法：
```
θ = θ + α * ∇logπθ(a|s) * G
```

### 优势函数（Advantage Function）

优势函数衡量动作相对于平均价值的好坏：
```
A(s, a) = Q(s, a) - V(s)
```

### 带基线的策略梯度

使用基线减少方差：
```
∇J(θ) = E[∇logπθ(a|s) * (G - b(s))]
```

其中，`b(s)`是基线（通常使用状态价值函数V(s)）。

### Actor-Critic算法

Actor-Critic结合了策略梯度和价值函数：

1. **Actor（策略网络）**：学习策略，选择动作
2. **Critic（价值网络）**：估计价值，计算优势

**更新规则**：
```
Actor更新：θ = θ + α * ∇logπθ(a|s) * A(s, a)
Critic更新：w = w + β * (G - Vw(s))²
```

### 深度确定性策略梯度（DDPG）

DDPG是用于连续动作空间的Actor-Critic算法：

1. 使用确定性策略：μ(s)
2. 使用深度神经网络作为Actor和Critic
3. 使用经验回放和目标网络

### 近端策略优化（PPO）

PPO是目前最流行的强化学习算法之一：

**核心思想**：限制策略更新的幅度，防止策略突变

**PPO-Clip目标函数**：
```
L(θ) = E[min(r(θ)A, clip(r(θ), 1-ε, 1+ε)A)]
```

其中，`r(θ) = πθ(a|s) / πθ_old(a|s)`是新旧策略的比值。

## 模型强化学习

### 什么是模型强化学习？

模型强化学习先学习环境模型，再使用模型规划最优策略。

### 环境模型

环境模型预测状态转移和奖励：
```
s' = f(s, a)
r = g(s, a)
```

### 基于模型的算法

- **Dyna-Q**：结合模型学习和无模型学习
- **MuZero**：AlphaZero的改进版，无需知道游戏规则

## 探索策略

### ε-贪心

最简单的探索策略，已在Q-learning中介绍。

### Boltzmann探索

根据Q值的softmax概率选择动作：
```
π(a|s) = exp(Q(s, a)/τ) / Σₐ' exp(Q(s, a')/τ)
```

其中，`τ`是温度参数：
- τ大：均匀随机选择
- τ小：倾向于选择Q值最大的动作

### Upper Confidence Bound（UCB）

UCB在多臂老虎机问题中使用：
```
UCB(a) = Q(a) + c * √(log(t) / N(a))
```

其中，`N(a)`是动作a被选择的次数，`c`是探索常数。

## 奖励设计

### 奖励工程

设计合适的奖励函数是强化学习的关键：

- **稀疏奖励**：只有完成目标才给奖励
- **稠密奖励**：每一步都给奖励

### 奖励塑造

通过修改奖励函数来引导学习：
- **势能塑造**：R'(s, a, s') = R(s, a, s') + γΦ(s') - Φ(s)
- **课程学习**：从简单任务开始，逐步增加难度

## 评估指标

### 平均回报（Average Return）

每个episode的平均回报：
```
Average Return = (G₁ + G₂ + ... + Gₙ) / n
```

### 成功率（Success Rate）

完成任务的episode占总episode的比例：
```
Success Rate = 成功次数 / 总次数
```

### 学习曲线（Learning Curve）

显示随训练步数变化的性能曲线。
