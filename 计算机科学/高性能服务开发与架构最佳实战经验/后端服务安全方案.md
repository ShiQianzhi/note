# 后端服务安全方案

## 一、安全架构总览

### 1.1 安全防护层次

```
┌─────────────────────────────────────────────────────────────┐
│                    安全防护层次                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐  │
│  │   WAF/网关   │    │   API网关    │    │   应用层     │  │
│  │  (边缘防护)  │    │  (认证限流)  │    │  (业务安全)  │  │
│  └──────┬───────┘    └──────┬───────┘    └──────┬───────┘  │
│         │                   │                   │           │
│         ▼                   ▼                   ▼           │
│  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐  │
│  │  DDoS防护    │    │   认证授权   │    │  输入验证    │  │
│  │  速率限制    │    │  令牌管理    │    │  参数校验    │  │
│  └──────────────┘    └──────────────┘    └──────────────┘  │
│                                                             │
│  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐  │
│  │   数据层     │    │   基础设施   │    │   运维安全   │  │
│  │  (加密存储)  │    │  (网络隔离)  │    │  (日志审计)  │  │
│  └──────────────┘    └──────────────┘    └──────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 核心安全原则

| 原则 | 说明 | 实施要点 |
|------|------|---------|
| **纵深防御** | 多层防护，单点失效不影响整体 | WAF + API网关 + 应用层验证 |
| **最小权限** | 只授予必要权限 | 细粒度角色权限、临时Token |
| **零信任** | 不信任任何外部请求 | 每次请求都需验证身份 |
| **数据加密** | 敏感数据全程加密 | 传输加密、存储加密、密钥管理 |
| **安全审计** | 完整记录所有操作 | 操作日志、访问日志、审计追踪 |

## 二、认证与授权

### 2.1 常见认证方式

| 方式 | 适用场景 | 安全性 | 复杂度 |
|------|---------|--------|--------|
| **API Key** | 服务器间通信 | 中 | 低 |
| **JWT** | 用户认证、单点登录 | 高 | 中 |
| **OAuth 2.0** | 第三方授权 | 高 | 高 |
| **OpenID Connect** | 身份认证协议 | 高 | 高 |
| **Basic Auth** | 内部服务、测试环境 | 低 | 低 |

### 2.2 JWT 实现

**生成 JWT**：

```go
package auth

import (
	"time"

	"github.com/golang-jwt/jwt/v5"
)

var secretKey = []byte("your-256-bit-secret")

type Claims struct {
	UserID   string `json:"user_id"`
	Username string `json:"username"`
	Role     string `json:"role"`
	jwt.RegisteredClaims
}

func GenerateToken(userID, username, role string) (string, error) {
	claims := Claims{
		UserID:   userID,
		Username: username,
		Role:     role,
		RegisteredClaims: jwt.RegisteredClaims{
			ExpiresAt: jwt.NewNumericDate(time.Now().Add(time.Hour * 24)),
			IssuedAt:  jwt.NewNumericDate(time.Now()),
			Issuer:    "api.example.com",
		},
	}

	token := jwt.NewWithClaims(jwt.SigningMethodHS256, claims)
	return token.SignedString(secretKey)
}

func ValidateToken(tokenString string) (*Claims, error) {
	token, err := jwt.ParseWithClaims(tokenString, &Claims{}, func(token *jwt.Token) (interface{}, error) {
		return secretKey, nil
	})

	if err != nil {
		return nil, err
	}

	if claims, ok := token.Claims.(*Claims); ok && token.Valid {
		return claims, nil
	}

	return nil, jwt.ErrInvalidClaim
}
```

**Gin 中间件**：

```go
package middleware

import (
	"net/http"
	"strings"

	"github.com/gin-gonic/gin"
	"github.com/example/api/auth"
)

func AuthMiddleware() gin.HandlerFunc {
	return func(c *gin.Context) {
		authHeader := c.GetHeader("Authorization")
		if authHeader == "" {
			c.JSON(http.StatusUnauthorized, gin.H{"error": "Authorization header required"})
			c.Abort()
			return
		}

		parts := strings.Split(authHeader, " ")
		if len(parts) != 2 || parts[0] != "Bearer" {
			c.JSON(http.StatusUnauthorized, gin.H{"error": "Invalid authorization format"})
			c.Abort()
			return
		}

		claims, err := auth.ValidateToken(parts[1])
		if err != nil {
			c.JSON(http.StatusUnauthorized, gin.H{"error": "Invalid token"})
			c.Abort()
			return
		}

		c.Set("user_id", claims.UserID)
		c.Set("username", claims.Username)
		c.Set("role", claims.Role)
		c.Next()
	}
}
```

### 2.3 权限控制

**基于角色的访问控制（RBAC）**：

```go
package auth

import (
	"net/http"

	"github.com/gin-gonic/gin"
)

func RequireRole(roles ...string) gin.HandlerFunc {
	return func(c *gin.Context) {
		userRole, exists := c.Get("role")
		if !exists {
			c.JSON(http.StatusForbidden, gin.H{"error": "Permission denied"})
			c.Abort()
			return
		}

		for _, role := range roles {
			if userRole == role {
				c.Next()
				return
			}
		}

		c.JSON(http.StatusForbidden, gin.H{"error": "Permission denied"})
		c.Abort()
	}
}
```

**使用示例**：

```go
func SetupRoutes() *gin.Engine {

	api.Use(middleware.AuthMiddleware())
	
	api.GET("/users", middleware.RequireRole("admin"), GetUsersHandler)
	api.GET("/profile", GetProfileHandler)
	
	return r
}
```

## 三、输入验证

### 3.1 常见攻击类型

| 攻击类型 | 说明 | 示例 |
|---------|------|------|
| **SQL注入** | 通过恶意SQL语句获取或修改数据 | `' OR 1=1--` |
| **XSS** | 注入恶意脚本 | `<script>alert('xss')</script>` |
| **命令注入** | 执行系统命令 | `; rm -rf /` |
| **路径遍历** | 访问非授权文件 | `../../etc/passwd` |
| **参数篡改** | 修改请求参数 | 将 `amount=1` 改为 `amount=-100` |

### 3.2 验证框架

**使用 go-playground/validator**：

```go
package handler

import (
	"net/http"

	"github.com/gin-gonic/gin"
	"github.com/go-playground/validator/v10"
)

var validate = validator.New()

type CreateUserRequest struct {
	Username string `json:"username" validate:"required,min=3,max=50,alphanum"`
	Email    string `json:"email" validate:"required,email"`
	Password string `json:"password" validate:"required,min=8,max=100"`
	Age      int    `json:"age" validate:"gte=18,lte=120"`
}

func CreateUserHandler(c *gin.Context) {
	var req CreateUserRequest
	if err := c.ShouldBindJSON(&req); err != nil {
		c.JSON(http.StatusBadRequest, gin.H{"error": err.Error()})
		return
	}

	if err := validate.Struct(req); err != nil {
		var errors []string
		for _, e := range err.(validator.ValidationErrors) {
			errors = append(errors, e.Field()+" is invalid")
		}
		c.JSON(http.StatusBadRequest, gin.H{"errors": errors})
		return
	}

	c.JSON(http.StatusOK, gin.H{"status": "success"})
}
```

### 3.4 SQL 注入防护

**使用预编译语句**：

```go
package repository

import (
	"database/sql"

	_ "github.com/lib/pq"
)

type UserRepository struct {
	db *sql.DB
}

func (r *UserRepository) GetUserByID(userID string) (*User, error) {
	var user User
	query := "SELECT id, username, email FROM users WHERE id = $1"
	err := r.db.QueryRow(query, userID).Scan(&user.ID, &user.Username, &user.Email)
	if err != nil {
		return nil, err
	}
	return &user, nil
}
```

## 四、敏感数据保护

### 4.1 密码安全

**密码哈希**：

```go
package auth

import (
	"golang.org/x/crypto/bcrypt"
)

func HashPassword(password string) (string, error) {
	bytes, err := bcrypt.GenerateFromPassword([]byte(password), bcrypt.DefaultCost)
	return string(bytes), err
}

func CheckPasswordHash(password, hash string) bool {
	err := bcrypt.CompareHashAndPassword([]byte(hash), []byte(password))
	return err == nil
}
```

**登录处理**：

```go
func LoginHandler(c *gin.Context) {

		Username string `json:"username"`
		Password string `json:"password"`

		c.JSON(http.StatusBadRequest, gin.H{"error": "Invalid request"})
		return

}
```

### 4.3 数据存储加密

**敏感字段加密**：

```go
package crypto

import (
	"crypto/aes"
	"crypto/cipher"
	"crypto/rand"
	"encoding/base64"
	"io"
)

type AESEncryptor struct {
	key []byte
}

func NewAESEncryptor(key []byte) *AESEncryptor {
	return &AESEncryptor{key: key}
}

func (e *AESEncryptor) Encrypt(plaintext string) (string, error) {
	block, err := aes.NewCipher(e.key)
	if err != nil {
		return "", err
	}

	gcm, err := cipher.NewGCM(block)
	if err != nil {
		return "", err
	}

	nonce := make([]byte, gcm.NonceSize())
	if _, err = io.ReadFull(rand.Reader, nonce); err != nil {
		return "", err
	}

	ciphertext := gcm.Seal(nonce, nonce, []byte(plaintext), nil)
	return base64.StdEncoding.EncodeToString(ciphertext), nil
}

func (e *AESEncryptor) Decrypt(ciphertext string) (string, error) {
	data, err := base64.StdEncoding.DecodeString(ciphertext)
	if err != nil {
		return "", err
	}

	block, err := aes.NewCipher(e.key)
	if err != nil {
		return "", err
	}

	gcm, err := cipher.NewGCM(block)
	if err != nil {
		return "", err
	}

	nonceSize := gcm.NonceSize()
	nonce, ciphertextData := data[:nonceSize], data[nonceSize:]

	plaintext, err := gcm.Open(nil, nonce, ciphertextData, nil)
	if err != nil {
		return "", err
	}

	return string(plaintext), nil
}
```

## 五、攻击防护

### 5.1 CSRF 防护

**生成 CSRF Token**：

```go
package csrf

import (
	"crypto/rand"
	"encoding/base64"
	"net/http"

	"github.com/gin-gonic/gin"
	"github.com/patrickmn/go-cache"
)

var csrfCache = cache.New(5*60, 10*60)

func GenerateToken() string {
	b := make([]byte, 32)
	rand.Read(b)
	token := base64.StdEncoding.EncodeToString(b)
	csrfCache.Set(token, true, 5*60)
	return token
}

func CSRFMiddleware() gin.HandlerFunc {
	return func(c *gin.Context) {
		if c.Request.Method == http.MethodPost || 
		   c.Request.Method == http.MethodPut || 
		   c.Request.Method == http.MethodDelete {
			
			token := c.GetHeader("X-CSRF-Token")
			if token == "" {
				c.JSON(http.StatusForbidden, gin.H{"error": "CSRF token required"})
				c.Abort()
				return
			}

			if _, exists := csrfCache.Get(token); !exists {
				c.JSON(http.StatusForbidden, gin.H{"error": "Invalid CSRF token"})
				c.Abort()
				return
			}

			csrfCache.Delete(token)
		}
		c.Next()
	}
}
```

### 5.3 路径遍历防护

```go
package security

import (
	"path/filepath"
	"strings"
)

func SafePath(basePath, userPath string) (string, error) {
	fullPath := filepath.Join(basePath, userPath)
	
	if !strings.HasPrefix(fullPath, filepath.Clean(basePath)+string(filepath.Separator)) && 
	   fullPath != filepath.Clean(basePath) {
		return "", fmt.Errorf("invalid path")
	}
	
	return fullPath, nil
}
```

**使用示例**：

```go
func GetFileHandler(c *gin.Context) {
	filePath := c.Param("path")
	
	safePath, err := SafePath("/data/files", filePath)
	if err != nil {
		c.JSON(http.StatusBadRequest, gin.H{"error": "Invalid path"})
		return
	}
	
	c.File(safePath)
}
```

### 6.1 环境变量管理

```go
package config

import (
	"fmt"
	"os"

	"github.com/joho/godotenv"
)

type Config struct {
	DBHost     string
	DBPort     string
	DBUser     string
	DBPassword string
	JWTSecret  string
	Env        string
}

func LoadConfig() (*Config, error) {
	if err := godotenv.Load(); err != nil && os.Getenv("ENV") != "production" {
		return nil, err
	}

	return &Config{
		DBHost:     os.Getenv("DB_HOST"),
		DBPort:     os.Getenv("DB_PORT"),
		DBUser:     os.Getenv("DB_USER"),
		DBPassword: os.Getenv("DB_PASSWORD"),
		JWTSecret:  os.Getenv("JWT_SECRET"),
		Env:        os.Getenv("ENV"),
	}, nil
}

func ValidateConfig(cfg *Config) error {
	if cfg.JWTSecret == "" {
		return fmt.Errorf("JWT_SECRET is required")
	}
	if len(cfg.JWTSecret) < 32 {
		return fmt.Errorf("JWT_SECRET must be at least 32 characters")
	}
	return nil
}
```

```go
func SecurityLogger() gin.HandlerFunc {

				c.ClientIP(),
				c.Request.Method,
				c.Request.URL.Path,
```

## 八、安全测试

### 8.1 单元测试

```go
package auth

import (
	"testing"
)

func TestHashPassword(t *testing.T) {
	password := "secure_password_123"
	hash, err := HashPassword(password)
	if err != nil {
		t.Fatalf("Failed to hash password: %v", err)
	}

	if len(hash) == 0 {
		t.Error("Hash should not be empty")
	}

	if !CheckPasswordHash(password, hash) {
		t.Error("Password verification failed")
	}

	if CheckPasswordHash("wrong_password", hash) {
		t.Error("Wrong password should not match")
	}
}

func TestJWT(t *testing.T) {
	token, err := GenerateToken("123", "testuser", "admin")
	if err != nil {
		t.Fatalf("Failed to generate token: %v", err)
	}

	claims, err := ValidateToken(token)
	if err != nil {
		t.Fatalf("Failed to validate token: %v", err)
	}

	if claims.UserID != "123" {
		t.Errorf("Expected user_id 123, got %s", claims.UserID)
	}

	if claims.Role != "admin" {
		t.Errorf("Expected role admin, got %s", claims.Role)
	}
}
```

## 十、常见安全错误

### 10.1 错误示例

```go
// 错误：明文存储密码
type User struct {
	Password string `json:"password"`
}

// 错误：硬编码密钥
var secretKey = "my-secret-key"

// 错误：直接拼接SQL
query := "SELECT * FROM users WHERE id = " + userID

// 错误：信任用户输入
filePath := "/data/" + userInput

// 错误：使用弱加密算法
hash := md5.Sum([]byte(password))
```

### 10.2 正确做法

```go
// 正确：哈希存储密码
type User struct {
	PasswordHash string `json:"-"`
}

// 正确：使用环境变量
secretKey := os.Getenv("JWT_SECRET")

// 正确：使用预编译语句
query := "SELECT * FROM users WHERE id = $1"

// 正确：验证路径
safePath := filepath.Join("/data", userInput)

// 正确：使用强加密算法
hash, _ := bcrypt.GenerateFromPassword([]byte(password), bcrypt.DefaultCost)
```

## 十一、总结

### 11.1 安全防护要点

1. **纵深防御**：多层防护机制
2. **最小权限**：只授予必要权限
3. **数据加密**：传输和存储都要加密
4. **输入验证**：不信任任何外部输入
5. **安全审计**：完整记录所有操作

### 11.2 持续改进

安全是一个持续的过程，需要：
- 定期进行安全审计和渗透测试
- 关注安全漏洞公告，及时修复
- 培训开发人员安全意识
- 建立安全开发规范和流程
