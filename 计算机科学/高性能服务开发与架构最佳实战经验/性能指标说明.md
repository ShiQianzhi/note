# 后端服务性能指标体系

## 一、核心指标框架

### RED 方法（面向请求）

| 指标 | 说明 | 关注重点 |
|------|------|---------|
| **Rate（速率）** | 每秒请求数（RPS/QPS） | 系统吞吐量 |
| **Errors（错误）** | 错误率（5xx/4xx） | 服务健康度 |
| **Duration（耗时）** | 响应时间分布（P50/P95/P99/P999） | 用户体验 |

### USE 方法（面向资源）

| 指标 | 说明 | 关注重点 |
|------|------|---------|
| **Utilization（利用率）** | 资源忙时占比（0-100%） | 资源饱和预警 |
| **Saturation（饱和度）** | 资源队列长度或等待时间 | 性能瓶颈识别 |
| **Errors（错误）** | 资源层面的错误计数 | 故障定位 |

## 二、延迟指标（Latency）

### 百分位数体系

| 指标 | 含义 | 应用场景 |
|------|------|---------|
| **P50 / Median** | 50% 请求的响应时间 ≤ 此值 | 常规性能基线 |
| **P90** | 90% 请求的响应时间 ≤ 此值 | 用户体验阈值 |
| **P95** | 95% 请求的响应时间 ≤ 此值 | SLA 承诺基准 |
| **P99** | 99% 请求的响应时间 ≤ 此值 | 关键业务体验保障 |
| **P999** | 99.9% 请求的响应时间 ≤ 此值 | 极端延迟监控 |
| **P9999** | 99.99% 请求的响应时间 ≤ 此值 | 金融级系统要求 |
| **Max** | 最大响应时间 | 异常检测 |

### 细分延迟指标

| 指标 | 说明 |
|------|------|
| **TTFB（Time To First Byte）** | 首字节时间，衡量服务端处理+网络传输的初始延迟 |
| **TTLB（Time To Last Byte）** | 末字节时间，完整响应时间 |
| **RTT（Round Trip Time）** | 往返时间，网络延迟 |
| **Processing Time** | 服务端处理时间（TTFB - RTT/2） |

## 三、吞吐量指标（Throughput）

### 请求级吞吐量

| 指标 | 全称 | 适用场景 | 备注 |
|------|------|---------|------|
| **QPS** | Queries Per Second | 数据库、Redis | 每秒查询数，原始含义 |
| **RPS** | Requests Per Second | HTTP 服务、API 网关 | 每秒请求数，HTTP 场景更准确 |
| **TPS** | Transactions Per Second | 业务事务 | 每秒完成的业务事务数 |

> **行业惯例**：在 HTTP 服务场景中，QPS 常被泛化为"每秒请求数"，与 RPS 混用。建议使用 **RPS** 表示 HTTP 请求数，**QPS** 表示数据库查询数。

### 数据级吞吐量

| 指标 | 说明 |
|------|------|
| **Throughput** | 单位时间处理的数据量（字节/秒） |
| **IOPS** | 每秒 I/O 操作次数（磁盘/网络） |

---

## 四、并发指标（Concurrency）

| 指标 | 说明 |
|------|------|
| **Concurrent Users** | 同时在线用户数 |
| **Active Connections** | 活跃连接数 |
| **In-flight Requests** | 正在处理中的请求数 |
| **Connection Pool Utilization** | 连接池利用率（目标 < 80%） |
| **Maximum Concurrent Requests** | 系统稳定处理的最大并发数 |

### 计算公式

```
并发数 = RPS × 平均响应时间（秒）
系统容量 = 单节点 RPS × 节点数 × 冗余系数
```

## 五、资源利用率指标

### 主机资源

| 指标 | 说明 | 告警阈值 |
|------|------|---------|
| **CPU Usage** | CPU 利用率 | 持续 > 80% |
| **CPU Saturation** | CPU 就绪队列长度 | > vCPU 核数的 10% |
| **Memory Usage** | 内存使用率 | > 85% |
| **Swap Usage** | 交换分区使用量 | > 0（OOM 预警） |
| **Disk I/O** | 磁盘读写吞吐量和 IOPS | 接近设备上限 |
| **Disk Utilization** | 磁盘繁忙率 | > 80% |
| **Network I/O** | 网络带宽利用率 | > 85% |
| **File Descriptors** | 文件描述符使用量 | > 80% 上限 |

### JVM 资源（Java 服务）

| 指标 | 说明 |
|------|------|
| **Heap Usage** | 堆内存使用率 |
| **GC Pause Time** | GC 停顿时间（P99 < 100ms） |
| **GC Frequency** | GC 触发频率 |
| **Thread Count** | 线程数 |
| **Deadlocks** | 死锁数量 |

## 六、可用性与可靠性指标

### 可用性

| 指标 | 说明 | 计算公式 |
|------|------|---------|
| **Uptime** | 系统正常运行时间百分比 | `(总时间 - 宕机时间) / 总时间 × 100%` |
| **Availability SLA** | 服务等级协议承诺 | 如 99.9%、99.99%、99.999% |

### 错误率

| 指标 | 说明 | 告警阈值 |
|------|------|---------|
| **Error Rate** | 错误请求占比 | > 1% |
| **5xx Rate** | 服务端错误率 | > 0.1% |
| **4xx Rate** | 客户端错误率 | > 5% |
| **Timeout Rate** | 超时请求率 | > 0.1% |

### 可靠性

| 指标 | 说明 |
|------|------|
| **MTBF** | Mean Time Between Failures，平均无故障时间 |
| **MTTR** | Mean Time To Recovery，平均恢复时间 |
| **MTTD** | Mean Time To Detect，平均检测时间 |

## 七、数据库与缓存指标

### 数据库

| 指标 | 说明 |
|------|------|
| **Query Latency** | 数据库查询延迟（P50/P95/P99） |
| **Slow Query Rate** | 慢查询率（目标 < 1%） |
| **Lock Wait Time** | 锁等待时间 |
| **Lock Contention** | 锁竞争率 |

| **Replica Lag** | 主从复制延迟 |
| **Cache Hit Rate** | 数据库缓存命中率（目标 > 95%） |

### 缓存（Redis/Memcached）

| 指标 | 说明 | 目标值 |
|------|------|--------|
| **QPS** | 每秒查询数 | - |
| **Hit Rate** | 缓存命中率 | > 90% |
| **Miss Rate** | 缓存未命中率 | < 10% |
| **Eviction Rate** | 缓存淘汰率 | < 1% |
| **Memory Usage** | 内存使用率 | < 80% |
| **Connection Count** | 连接数 | < 连接池上限 |

## 十一、性能测试流程

1. **基准测试（Baseline）**：建立系统性能基线
2. **负载测试（Load Testing）**：模拟正常业务负载
3. **压力测试（Stress Testing）**：逐步增加负载直到系统崩溃
4. **稳定性测试（Soak Testing）**：长时间运行验证稳定性
5. **容量规划（Capacity Planning）**：根据测试结果规划资源
6. **回归测试（Regression Testing）**：代码变更后验证性能

## 十二、总结

### 指标选择指南

| 场景 | 核心指标 |
|------|---------|
| **用户体验** | P99 延迟、错误率 |
| **系统容量** | RPS、并发数 |
| **资源瓶颈** | CPU/内存/磁盘利用率、饱和度 |
| **SLA 承诺** | P95/P99 延迟、可用性 |
| **数据库性能** | 查询延迟、慢查询率、缓存命中率 |

### 黄金指标框架

衡量后端性能的核心是**建立可观测性体系**，通过多维度指标的组合分析来定位瓶颈：

- **延迟（Latency）**：系统响应速度
- **流量（Traffic）**：系统处理的请求量
- **错误（Errors）**：失败请求的比例
- **饱和度（Saturation）**：资源使用程度
